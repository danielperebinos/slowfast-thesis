# Architecture

The thesis codebase is a **modular monolith** organized around a shared kernel (`experiments/core/`) and N independent experiment modules (`experiments/experiment_0N/`).

This page is the short, developer-facing mirror of [`.ai-factory/ARCHITECTURE.md`](../.ai-factory/ARCHITECTURE.md) — read that file for full rationale, forbidden-pattern examples, and code samples. This page answers: "where does my code go?"

## Directory Map

```
experiments/
├── core/                 # shared kernel — variant-agnostic
│   ├── data_loader.py    # AvaAnticipationDataset, PackPathway, transforms, factory
│   ├── loss.py           # ActionAnticipationLoss (BCE + exp temporal penalty)
│   └── metrics.py        # multilabel_accuracy, topk_accuracy, AverageMeter, calculate_tta
├── dataset/              # one-time data prep (outside the training loop)
│   ├── build_splits.py   # AVA CSV + local videos → train.csv / test.csv
│   ├── download_all.py   # parallel yt-dlp + ffmpeg fetch
│   └── fetch.py          # single-video fetch helpers
├── experiment_01/        # Baseline SlowFast R50
├── experiment_02/        # + Non-Local / Self-Attention
├── experiment_03/        # + YOLOv8 ROI mask on Fast pathway
├── experiment_04/        # Hybrid ROI + local attention
└── verify_setup.py       # dummy-data forward/backward smoke test
```

## Dependency Rules

The module graph must stay a DAG with `experiments/core/` as the single sink.

| From | To | Allowed? |
|------|----|----------|
| `experiments/experiment_0N/*` | `experiments.core.*` | ✅ yes — every variant imports the kernel. |
| `experiments/dataset/*` | `experiments.core.*` | ✅ yes — data prep can reuse label-map logic. |
| `experiments/core/*` | PyTorch / PyTorchVideo / stdlib | ✅ yes — the kernel is pure infrastructure. |
| `experiments/core/*` | any `experiment_0N` | ❌ no — the kernel MUST NOT know about variants. |
| `experiment_0N/*` | `experiment_0M/*` | ❌ no — cross-variant imports break fairness. |
| `experiments/core/*` | `experiments/dataset/*` | ❌ no — at training time CSVs already exist. |

## Variant Module Layout

Each `experiment_0N/` is self-contained:

```
experiment_0N/
├── __init__.py
├── train.py              # self-contained entry point
├── model.py              # variant-specific architecture (experiments 02-04)
├── train.csv / test.csv  # split CSVs (generated by experiments/dataset/build_splits.py)
├── checkpoint.pth        # resume-state — PROTECTED
└── best_<variant>.pth    # best-on-val weights — PROTECTED
```

## The Shared Training Script Shape

Every `experiment_0N/train.py` follows the same skeleton:

1. `sys.path.append(...)` to the project root so `from experiments.core import ...` resolves.
2. Constants block: `EPOCHS`, `BATCH_SIZE`, `LR`, `NUM_WORKERS`, plus path overrides via `os.getenv` (`VIDEO_DIR`, `TRAIN_CSV`, `VAL_CSV`, `CKPT_PATH`).
3. `get_model(num_classes)` — builds the variant-specific network.
4. `train_epoch` / `validate` — use `AverageMeter` for loss/acc, print every 10 steps.
5. `save_checkpoint` / `load_checkpoint` — persist `epoch, model, optimizer, scheduler, best_val_acc, label_map`.
6. `main()` — builds `label_map` once from `TRAIN_CSV`, restores checkpoint if present, logs to MLflow under the `SlowFast_Anticipation` experiment with a variant-scoped run name.

Keep new variants consistent with this shape. Deviations are acceptable when justified; copies of this skeleton that drift silently are not.

## Key Principles

1. **Shared kernel is sacred.** Code that must be identical across variants — dataset, loss, metrics — lives in `experiments/core/`. No forking "just this one tweak".
2. **Variants are independent processes.** Run them directly (`python experiments/experiment_0N/train.py`). No parent orchestrator today; this keeps MLflow run lifecycles independent and failures isolated.
3. **Configuration is in code + env vars, not a DI container.** Hyperparameters are module-level constants; paths are `os.getenv(...)` with defaults. A thesis examiner must be able to verify the pipeline end-to-end at a glance.
4. **Reproducibility over abstraction.** Fixed seed, deterministic val transforms, one label map shared between train/val. Any abstraction that hides the data pipeline is a bug.
5. **MLflow is an integration point, not a wrapper.** Call `mlflow.log_*` directly. Do not introduce a `Tracker` class unless it pays for itself across many more variants than we have.

## Anti-Patterns (Short List)

- ❌ Copying `data_loader.py` into a variant folder to tweak a knob. Parameterize the factory instead.
- ❌ `if variant == "...":` branches inside `core/`. The kernel must remain variant-agnostic.
- ❌ `from experiments.experiment_02.train import ...` from another variant. Extract to `core/` and import from both.
- ❌ Touching Docker volumes to "clean up" between runs. Use MLflow run filters or a fresh experiment name. Never delete `volumes/postgres` or `volumes/minio`.
- ❌ Module-level `mlflow.set_experiment(...)` — mutates run state at import time. Keep MLflow calls inside `main()`.
